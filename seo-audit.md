# Xudumela — SEO foundations audit

One-line summary: **the site is invisible to Google and to social platforms.** Zero meta descriptions, zero Open Graph tags, zero structured data, and the sitemap referenced by robots.txt returns the homepage instead of an XML index. None of this is hard to fix, but none of it has been done.

## Metadata — per page

| URL | Title tag | Meta description | Canonical |
|-----|-----------|------------------|-----------|
| `/` | Xudumela | **MISSING** | `https://xudumela.africa/` |
| `/stay/` | Stay – Xudumela | **MISSING** | `https://xudumela.africa/stay/` |
| `/contact/` | Contact – Xudumela | **MISSING** | `https://xudumela.africa/contact/` |
| `/poachella-festival/` | Poachella / Festival – Xudumela | **MISSING** | `https://xudumela.africa/poachella-festival/` |
| `/room/camp-site/` | CAMP SITE – Xudumela | **MISSING** | `https://xudumela.africa/room/camp-site/` |
| `/room/safari-tent/` | SAFARI TENT – Xudumela | **MISSING** | `https://xudumela.africa/room/safari-tent/` |

**Title pattern:** `<Page Name> – Xudumela`. The homepage title is just `Xudumela` with no qualifier. Every other page uses the en-dash separator (`–`), which is fine for SEO but visually weak. No page mentions Okavango Delta, Maun, Botswana, or any commercial keyword in the title. Massive miss.

**Meta descriptions:** zero pages have them. Google will auto-generate a description from page content, and given how thin most pages are, the autogenerated snippets will be poor.

## Open Graph / Twitter Card tags

**Status: none of the six pages have any OG or Twitter tags.**

Practical consequences:
- A link to xudumela.africa pasted into WhatsApp, iMessage, Instagram DMs, Slack, or Twitter shows the URL only — no hero image, no description, no card.
- The Poachella festival page in particular shares badly. People will not click.
- Facebook/Meta sharing will fall back to a random in-page image.

## Robots directives

All pages have `<meta name="robots" content="max-image-preview:large" />`. No `noindex` or `nofollow` directives. Site is fully indexable. Good.

## Structured data / JSON-LD

**Status: zero JSON-LD blocks across all six pages.**

Schema types that should exist and don't:
- `Organization` or `LocalBusiness` on the homepage (with NAP, phone, email, geo coords)
- `LodgingBusiness` for the camp itself (with starsRating, amenityFeature, priceRange)
- `Campground` or `HotelRoom` × 2 for the two accommodation types
- `Event` or `MusicEvent` for Poachella 2026 (this is the biggest miss — event-rich results in Google would 3–5× this page's CTR)
- `BreadcrumbList` across the deeper pages
- `FAQPage` if the Poachella FAQ ever gets populated
- `WebSite` with `SearchAction` for the homepage

## Site infrastructure

| Resource | URL | Status |
|----------|-----|--------|
| `robots.txt` | `/robots.txt` | ✓ Valid. Blocks `/wp-admin/`. References sitemap at `/wp-sitemap.xml`. |
| WordPress XML sitemap | `/wp-sitemap.xml` | **BROKEN — returns the homepage HTML (200 OK, wrong content-type).** |
| Generic sitemap | `/sitemap.xml` | **BROKEN — returns the homepage HTML.** |
| RSS feed | `/feed/` | Works but the channel is empty (no posts). |
| OpenSearch description | — | Not present. |
| 404 behaviour | Not tested in this audit. |

**The broken sitemap is a real SEO bug.** robots.txt tells Google to crawl `/wp-sitemap.xml`. Google requests it, gets HTML back instead of XML, and gives up. Discoverability is then dependent entirely on internal linking and external backlinks — of which the site has very few.

Fix: install or activate a sitemap plugin (Yoast, Rank Math, or WordPress core's built-in sitemap which is what `wp-sitemap.xml` is supposed to serve). Then re-verify that the URL returns `Content-Type: application/xml`.

## Generator tags / stack exposure

Every page exposes the WordPress and Elementor versions in `<meta name="generator">`:

```
<meta name="generator" content="WordPress 6.9.4" />
<meta name="generator" content="Elementor 4.0.8; features: e_font_icon_svg, additional_custom_breakpoints; ..." />
```

This is fine for SEO but is a small security/maintenance signal. Worth removing in the rebuild.

## On-page SEO patterns

| Issue | Evidence |
|-------|----------|
| **Two H1s on homepage** | "Welcome to Xudumela" AND "Where Conservation meets Wilderness" both rendered as H1. Pick one. |
| **H3 used as paragraph** | "More than a place to stay, Xudumela is a living conservancy..." is wrapped in an H3 element on multiple pages. It should be a `<p>`. |
| **Image alt text** | Almost no image on the site has alt text. Out of the 43 unique non-logo images captured, only a couple of Elementor cropped thumbs have `alt="Screenshot"` set. Every other image has no alt. Major accessibility AND SEO miss. |
| **Internal linking depth** | Sparse. The four homepage "experiences" cards link to `#` placeholders. The "Explore All Accommodations" CTA links off-site to the theme demo. |
| **Breadcrumbs** | Not present visually or in schema. |
| **Heading hierarchy violations** | Multiple H6s used in the footer for what are clearly small-label widgets (XUDUMELA, Reach Out, Sign up for newsletter). Use `<p class="small-label">` or `<span>` styled equivalents instead. |

## Performance signals

| Signal | Status |
|--------|--------|
| Image formats | All JPG/PNG. **No WebP or AVIF served**, despite WordPress 6.9.4 supporting WebP natively. Largest images on the site are 7MB (`1660413.jpg`) and 6.5MB (`1660618.jpg`). |
| Largest images on the page | Multiple multi-MB hero images served at original resolution to mobile devices. |
| Font loading | **Severe over-loading.** Theme loads `Marcellus` and `Jost` (both used) plus `Roboto` and `Roboto Slab` (both unused), each with **all weights 100–900 plus italics**. That's roughly 36 font files for two visible families. |
| Resource hints in `<head>` | `dns-prefetch` to `fonts.googleapis.com` and `fonts.gstatic.com`. No `preload` on LCP image. No `preconnect` to image CDN. |
| Lazy-loading | Elementor handles below-the-fold images via its own lazyload. Above-the-fold hero image is not lazy-loaded. |

**Quick wins for rebuild:**
1. Drop Roboto + Roboto Slab entirely (Marcellus + Jost only)
2. Limit Marcellus and Jost to the two weights actually used (Jost 400 + 500, Marcellus 400)
3. Convert and serve all hero images as WebP, max 1920px wide, max 300KB each
4. Add `preload` for the LCP hero image
5. Add `loading="lazy"` to all below-the-fold images

## SEO priority fix list

In order of pain caused per minute of work:

1. **Fix the sitemap.** Install Yoast or Rank Math. Verify XML returns at `/wp-sitemap.xml`. (15 min)
2. **Add Open Graph + Twitter Card tags** to every page. Hero image, title, description per page. Without these the festival page is unshareable. (45 min)
3. **Write meta descriptions** for all six pages, each ~150 chars, mentioning Okavango / Maun / Botswana / camp / festival as relevant. (30 min)
4. **Add `Event` / `MusicEvent` JSON-LD** to the Poachella page. This alone could 3–5× the page's organic CTR via rich results. (30 min)
5. **Add `LocalBusiness` / `LodgingBusiness` JSON-LD** to the homepage. (20 min)
6. **Fix the broken homepage** — remove the duplicate H1, swap the H3-as-paragraph to a `<p>`, populate or remove the zero-count stat strip, fix the `#` placeholder CTAs, fix the off-site "Explore All Accommodations" link. (1 hr)
7. **Add alt text to every image.** (1 hr)
8. **Clean up NAP.** One phone, one email per intent, one address per location, one canonical contact block. Remove theme-demo placeholders. (15 min)
9. **Add an /about/ page.** Founders, partners, history, conservation track record. (drafting work)
10. **Convert hero images to WebP and right-size for mobile.** Drop unused Google Fonts. (2 hr)
